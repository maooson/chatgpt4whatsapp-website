## 登录失败

如果 Google 登录无法完成，优先检查：

- 浏览器是否拦截了弹窗
- 三方 Cookie 是否被禁用
- 当前 Chrome Profile 是否切错 Google 账号
- 页面是否停留在旧的缓存状态

建议动作：

1. 刷新 WhatsApp Web
2. 重新打开侧边栏
3. 再发起一次 Google 登录

## 绑定激活码失败

优先核查这几个点：

- 激活码是否完整复制
- 是否有多余空格
- 激活码是否已过期
- 设备激活数量是否达到上限

如果你购买时使用的邮箱和当前 Google 邮箱不同，也不一定是问题。只要 License 本身有效，仍然可以绑定。

## 付费功能突然被锁

这类问题通常与以下因素有关：

- 订阅已到期
- 最近一次扣款失败
- 订阅状态变成 `past_due`、`paused` 或 `expired`
- 后端还没有同步到最新订阅状态

这时建议先检查账号页和套餐状态，而不是直接重装扩展。

## AI 功能无结果或报错

如果 AI 分级、结构化线索或辅助回复失败，通常优先检查：

- API Key 是否正确
- 账户是否有可用额度
- 模型名称是否仍然有效
- 当前套餐是否支持该 Connector
- 公司网络是否拦截外部 API

## 定时消息没有发出去

先看任务列表中的状态，再检查：

- 联系人是否存在于本地客户库
- 模板里的变量是否正确
- 发送时手机和 WhatsApp Web 是否在线

## 联系支持前请准备什么

为了更快定位问题，建议提前提供：

1. 具体报错文案
2. 失败页面截图
3. 问题影响范围：一个账号还是所有账号
4. 问题类型：登录、绑定、AI、发送、权限

> [!IMPORTANT]
> 不要只说“不能用了”。把出错页面、时间点和具体操作步骤一起发给支持，排查速度会快很多。
